# キット混色コンポーザー: 2段階フロー・種別絞り込み・キーボード追従 設計

## 背景・目的

`docs/superpowers/specs/2026-07-13-kit-color-mixing-design.md`で実装した`KitColorComposerModal`(色名入力+塗料選択+割合編集が1画面に同居)を実機で使ったユーザーから、以下の改善要望が出た:

1. 色名と「塗料種別」(ラッカー/水性アクリル/エナメル/エマルジョン)を先に決めてから塗料選択画面に遷移する2段階フローにしたい
2. アコーディオン内の割合%入力時、オンスクリーンキーボードが入力欄を隠して操作しづらい
3. 塗料種別をまたいだ物理的な混色はできないため、選択した種別の塗料のみを候補として表示してほしい

要望1と3は密接に関係する: 塗料種別を先に決めることで、その後の塗料選択(一覧ドリルダウン・近似色検索)を同種別のみに絞り込める。

## スコープ

### 今回変更するもの

- `KitColorComposerModal.tsx`を「設定」(色名+塗料種別)→「塗料選択」(既存のタブ+アコーディオン)の2ステップ構成に変更
- `HierarchyBrowser.tsx`に任意prop`paintType`を追加し、指定時はブランド/シリーズ/塗料の3クエリすべてを種別で絞り込む
- `ColorMatcher.tsx`に任意prop`lockedPaintType`を追加し、指定時は種別フィルタをその1種類に固定して種別チップ行を非表示にする
- 塗料選択ステップの画面全体を`KeyboardAvoidingView`で包み、割合入力欄がキーボードに隠れないようにする

### 今回もやらないこと

- `HierarchyBrowser`/`ColorMatcher`の`AddPaintModal`側での既存の挙動変更(両propとも未指定時は現状のまま、無変更)
- 塗料種別が未設定(`paint_type IS NULL`、全体の約2%)の塗料をこの機能で選択可能にすること — 種別で絞り込む以上、種別不明の塗料は対象外のままとする
- 割合編集を独立した3ステップ目に分離すること — 「塗料を選びながらリアルタイムに混色プレビューを確認したい」という当初の要件を維持するため、割合編集は塗料選択ステップに残す

## 画面フロー

```
[ステップA: 設定]                    [ステップB: 塗料選択]
┌─────────────────┐         ┌─────────────────┐
│ 色名 [_______]    │  次へ→  │ ← (戻ると選択済み塗料は │
│ 塗料種別:          │         │    クリアされる)      │
│ [ラッカー][水性アクリル]│         │ [一覧から][近似色検索]  │
│ [エナメル][エマルジョン]│         │  (種別で絞り込み済み)  │
└─────────────────┘         │ ▼現在の色 ■■■         │
                            │ [展開時、割合編集]      │
                            │         [保存]        │
                            └─────────────────┘
```

- ステップAの「次へ」は塗料種別が選択されるまで非活性
- ステップBのヘッダーに戻るボタンを追加。押すとステップAへ戻り、選択済み塗料は破棄する(種別を変えた場合の不整合を防ぐため。色名の入力内容は保持)
- ステップB全体を`KeyboardAvoidingView`(iOS: `behavior="padding"`、Androidは`behavior`未指定でOS標準のリサイズに任せる)で包む

## `HierarchyBrowser.tsx`の変更

新規の任意prop `paintType?: string` を追加。指定時、以下の3クエリすべてに `AND paint_type = ?` を追加する:

- ブランド一覧: `SELECT DISTINCT brand FROM catalog_paints WHERE paint_type = ? ORDER BY brand`
- シリーズ一覧: `SELECT series, MAX(series_en) AS series_en FROM catalog_paints WHERE brand = ? AND paint_type = ? GROUP BY series ORDER BY series`
- 塗料一覧(`loadPaints`の`where`配列に条件追加): `paint_type = ?`

未指定時(`paintType === undefined`)は既存の`AddPaintModal`での利用と完全に同じ挙動を保つ。

## `ColorMatcher.tsx`の変更

新規の任意prop `lockedPaintType?: string` を追加。指定時:

- コンポーネントマウント時に`selectedTypes`を`[lockedPaintType]`で初期化
- 種別フィルタのチップ行(`typeOptions`の表示エリア)を丸ごと非表示にする(ユーザーが変更できないようにする)
- つや(gloss)フィルタは従来通り操作可能

未指定時は既存の`AddPaintModal`での利用と完全に同じ挙動を保つ。

## 検証方法

- `npx tsc --noEmit`
- UTF-8 BOMなし確認
- 実機での確認: キット詳細→色を追加→色名入力→塗料種別(例: ラッカー塗料)を選択→次へ→一覧ドリルダウンでラッカー塗料以外のブランド/シリーズ/塗料が出てこないことを確認。近似色検索タブでも同様に種別チップが出ず、ラッカー塗料のみが結果に出ることを確認。割合入力欄をタップし、キーボード表示時も入力欄が隠れず操作できることを確認。ステップBから戻るボタンでステップAへ戻り、種別を変更→再度次へで、以前選んでいた塗料がクリアされていることを確認。
